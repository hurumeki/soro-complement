# 算盤補数パズルゲーム 仕様書

**プロジェクト名:** そろばんチャレンジ  
**対象環境:** スマートフォン・タブレット ブラウザ（オフライン動作）  
**対象ユーザー:** 小学生  
**実装スタック:** HTML / CSS / JavaScript（シングルファイル）

---

## 1. ゲームコンセプト

画面上部に表示される数字（問題）に対して、算盤で**補数**を入力し、合計が「1桁上の10の倍数」になるように完成させるアクションパズルゲーム。
制限時間1分以内に何問クリアできるかを競い、ハイスコアを目指す。

### 補数とは
| 問題 | 作るべき補数 | 合計 |
|------|------------|------|
| 3    | 7          | 10   |
| 27   | 73         | 100  |
| 123  | 877        | 1000 |

## 3. 算盤の仕様

### 3.1 珠の構成（1軸あたり）

日本式算盤の構成に準拠する。

- **上珠（五珠）:** 1個、価値 = 5
- **下珠（一珠）:** 4個、価値 = 各1
- 梁（はり）に珠を寄せることで計上

| 上珠の状態 | 下珠の状態 | 軸の値 |
|-----------|-----------|--------|
| 梁から離れている | 0個寄せ | 0 |
| 梁から離れている | 1個寄せ | 1 |
| 梁から離れている | 2個寄せ | 2 |
| 梁から離れている | 3個寄せ | 3 |
| 梁から離れている | 4個寄せ | 4 |
| 梁に寄せている   | 0個寄せ | 5 |
| 梁に寄せている   | 1個寄せ | 6 |
| 梁に寄せている   | 2個寄せ | 7 |
| 梁に寄せている   | 3個寄せ | 8 |
| 梁に寄せている   | 4個寄せ | 9 |

### 3.2 操作方法

**タップ操作（推奨主操作）—「珠の間をタップで分割」方式:**

珠と珠の間をタップすると、その隙間を境にして**上側の珠は上へ、下側の珠は下へ**分かれて移動する。結果として、タップした位置で珠が上下に分割された状態になり、その位置の値が算盤に設定される。

**下珠エリアの操作例（軸の値が0の場合）:**

下珠は梁の下に4個、初期状態では全て下に寄っている。

```
初期状態（値 = 0）:
  [梁]
   ○  ← 下珠1（梁に最も近い）
   ○  ← 下珠2
   ○  ← 下珠3
   ○  ← 下珠4（最下部）

上から2つ目と3つ目の間をタップ:
  [梁]
   ○  ← 下珠1（梁に寄る）
   ○  ← 下珠2（梁に寄る）
   ─  ← ここをタップ
   ○  ← 下珠3（下に残る）
   ○  ← 下珠4（下に残る）
結果: 値 = 2
```

**操作の一般則:**
- 下珠エリアでタップした隙間より**上側**にある下珠はすべて梁に寄る
- タップした隙間より**下側**にある下珠はすべて下に離れる
- 上珠（五珠）エリアでは: 梁と上珠の間をタップ→上珠が梁に寄る（+5）、上珠の上をタップ→上珠が離れる（-5）

**珠の組み合わせ例（下珠のみ）:**

| タップ位置 | 梁に寄る下珠の数 | 値 |
|-----------|--------------|-----|
| 下珠1の上（梁との隙間） | 1個 | 1 |
| 下珠1と2の間 | 2個 | 2 |
| 下珠2と3の間 | 3個 ※ただし上の2個 | ※実際は上から数えてタップ位置までの個数 |
| 下珠3と4の間 | 3個 | 3 |
| 下珠4の下 | 4個 | 4 |

**上珠と下珠の複合（値5〜9）:** 上珠を梁に寄せた状態で下珠エリアを操作することで6〜9を表現。

**フリック操作（サポート）:**
- 各珠を上下にフリックすることで個別移動

**リセットボタン:**
- フッターの「クリア」ボタンで算盤を全軸0にリセット

### 3.3 算盤の初期状態
- ゲーム開始時・問題クリア時: 全軸 0

## 4. ゲームシステム

### 4.1 問題生成

- 桁数はゲーム開始時に決定（難易度選択または自動）
- 各桁の数字は1〜9のランダム値（先頭桁は必ず1以上）
- 初期問題数を生成してキューに格納し、足りなくなったら追加生成

### 4.2 加算ロジック

#### 加算タイミング
下位桁から上位桁に向かって**桁ごとに判定**する。

**1桁目（最下位）が問題桁と合算して10になったとき:**
- その桁のみ加算
- 問題数字の1桁目を繰り上げる（2桁目に+1）
- 算盤の1桁目はその値のまま**ロック（操作不可）**
- 問題数字が繰り上がったことをアニメーションで表示

**2桁目も合算して10になったとき（1桁目ロック済みの状態）:**
- 2桁目を加算
- 問題数字の2桁目を繰り上げる
- 算盤の2桁目も追加でロック
- 1桁目・2桁目はロックしたまま、プレイヤーは3桁目の操作のみ行う

**最上位桁（3桁目）も合算して10になったとき（下位桁すべてロック済みの状態）:**
- 全桁まとめてクリア判定
- 問題クリア演出を発火
- 算盤を全桁0にリセット（ロック解除）、次の問題へ

#### 具体的な例（3桁、問題: 123）

```
状態A: 算盤 = 007
  → 1桁目: 3 + 7 = 10 → 繰り上がり
  → 問題: 130 に更新
  → 算盤1桁目: ロック（7のまま）、2桁目・3桁目は操作可

状態B: 算盤 = 077（1桁目ロック中）
  → 2桁目: 3 + 7 = 10 → 繰り上がり
  → 問題: 200 に更新
  → 算盤2桁目: 追加でロック（7のまま）、3桁目のみ操作可

状態C: 算盤 = 877（3桁目のみ操作）
  → 3桁目: 2 + 8 = 10 → 全桁クリア！
  → 問題: 1000（クリア演出）
  → 算盤: 全桁0にリセット・ロック解除、次の問題へ
```

#### 操作順序によるペナルティ
- 下位桁が未完成なのに上位桁を操作しても**加算は発生しない**（珠は動くが問題数字に反映されない）
- 例: 1桁目を合わせる前に2桁目を8にしても、問題数字は変化しない

### 4.3 スコアシステム

#### 基本スコア
| 加算方法 | スコア |
|---------|--------|
| 1桁ずつ逐次クリア | 桁数 × 10点 |
| 2桁同時クリア | 2桁 × 15点（+50%ボーナス） |
| 3桁同時クリア | 3桁 × 20点（+100%ボーナス） |
| N桁同時クリア | N桁 × (10 + N×5)点 |

**同時クリアの定義:** 最上位桁の加算が行われた瞬間に下位桁も含めてすべて確定すること（最下位桁から順番ではなく、同一フレーム内で全桁が揃う）

#### ボーナス
- **スピードボーナス:** 問題表示からクリアまでの時間が短いほど加点（最大+50%）
- **連続クリアボーナス:** 連続してクリアするごとに係数アップ（最大 ×3）

#### ペナルティ
- ロック中の桁を誤操作しようとした場合: 演出（バイブ・赤点滅）のみ、スコア変化なし

### 4.4 難易度

| 難易度 | 桁数 | 問題の特徴 | 特記事項 |
|--------|------|-----------|---------|
| れんしゅう | 2桁 | 各桁1〜5 | 各桁の過不足をリアルタイム表示（後述） |
| やさしい | 2桁 | 各桁1〜9 | — |
| ふつう | 3桁 | 各桁1〜9 | — |
| むずかしい | 4桁 | 各桁1〜9 | — |
| チャレンジ | 5桁 | 各桁1〜9 | — |

#### 「れんしゅう」モードの特別表示

各軸の上部（問題数字の下）に、現在の算盤の値と補数の関係をリアルタイムで表示する。

| 算盤の値 vs 必要な補数 | 表示 |
|----------------------|------|
| 少ない（もっと増やす） | 青い下向き矢印 ▼ または「もっと」 |
| ちょうど合っている     | 緑のチェック ✓ |
| 多すぎる             | 赤い上向き矢印 ▲ または「すくない」 |

**目的:** 2桁目だけ合っていても、1桁目が未完成なら加算が起きないことを視覚的に理解させる。2桁目に✓が出ていても1桁目が▼のままなら加算されないことを示す。

## 5. 制限時間・ゲームフロー

```
[タイトル画面]
  → 難易度選択
  → [ゲーム画面] 3秒カウントダウン後スタート
  → 1分間プレイ
  → [タイムアップ演出]
  → [リザルト画面] スコア・最高スコア表示
  → [タイトルに戻る / もう一度]
```

### タイマー仕様
- 残り時間をプログレスバー＋数字で表示
- 残り10秒で警告演出（色変化・点滅）
- タイムアップと同時に算盤操作をロック

## 6. UI / UX 仕様

### 6.1 デザイン方針

- **テーマ:** 和風モダン。木目調の算盤フレーム＋現代的なフラットUIの融合
- **カラーパレット:**
  - 背景: 深い藍色（#1a2744）または濃いグリーン（#1a3320）
  - 算盤フレーム: 木目調のウォームブラウン
  - 珠: 象牙色〜クリーム色（#f5e6c8）、選択時はゴールドに光る
  - 梁: 濃いブラウン（#4a2c0a）
  - 問題数字: 白〜ライトゴールド、大きくくっきり
  - ロック済み桁: グレー表示

- **フォント:** 問題数字に太くて視認性の高い日本語対応フォント（例: Noto Sans JP Bold または Zen Kaku Gothic）

### 6.2 算盤の視覚表現

- 各珠は角丸の楕円形（リアルな算盤の珠のシルエット）
- 珠には光沢グラデーションを入れ、立体感を演出
- フレームには木目テクスチャ（CSSまたはSVGで表現）
- 軸は細い縦線で、珠が滑らかにスライド

### 6.3 アニメーション・演出

| イベント | 演出 |
|---------|------|
| 珠の移動 | 物理感のあるスライドアニメーション（ease-out、30〜60ms） |
| 桁繰り上がり | 問題数字が弾むようにカウントアップ、ピカっと光る |
| 桁ロック | その桁が半透明+グレーになり、ロックアイコン表示 |
| クリア | 全体に放射状のパーティクル爆発、「✓」マーク、効果音（Web Audio API） |
| 同時クリアボーナス | より派手なパーティクル＋「COMBO!」テキスト |
| タイムアップ | 画面フリーズ→フェードアウト |
| 残り10秒 | タイマーが赤く点滅 |
| ハイスコア更新 | キラキラアニメーション |

### 6.4 ハプティクスフィードバック
- 珠の移動時: `navigator.vibrate(10)`（軽いバイブ）
- クリア時: `navigator.vibrate([20, 30, 50])`（パターンバイブ）
- 操作不可時: `navigator.vibrate(100)`（長めのバイブ）

## 7. 効果音（Web Audio API）

オフライン対応のため、外部ファイル不使用。Web Audio APIでプログラマブルに生成。

| イベント | 音の種類 |
|---------|---------|
| 珠の移動 | 短いクリック音（木の衝突音風）|
| 繰り上がり | 上昇音（ピコ） |
| クリア | 達成音（明るいチャイム） |
| コンボクリア | より華やかなチャイム |
| タイムアップ | 終了音 |
| 操作不可 | 低いブザー |

## 9. 状態管理

```
GameState {
  phase: 'title' | 'countdown' | 'playing' | 'result'
  difficulty: 'practice' | 'easy' | 'normal' | 'hard' | 'challenge'
  
  // タイマー

}
```

## 12. 技術的な注意点

- **タッチイベント:** `touchstart` / `touchend` を使用、`preventDefault()` でスクロール防止
- **珠のヒット判定:** 珠の描画領域だけでなく、珠間の隙間も広めにタップ判定エリアとする
- **ゲームループ:** `requestAnimationFrame` でタイマー更新
- **レスポンシブ:** `vw` / `vh` 単位で算盤サイズを画面幅に合わせてスケール
- **iOS Safari対応:** `-webkit-tap-highlight-color: transparent` を設定し、ダブルタップズームを無効化
